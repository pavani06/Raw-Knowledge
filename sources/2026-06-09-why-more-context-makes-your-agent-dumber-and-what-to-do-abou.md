## Key Takeaways

- **More context makes agents dumber** — a meta-analysis shows accuracy drops from ~70% at 4K tokens to ~50% at 90K; context window size is not the solution, it's a challenge ([[concepts/context-window-management]])
- **Static prompts eat context** — every agent call carries a base prompt that takes a big chunk of the window; Google's Agents Hack paper recommends shorter prompts ([[concepts/prompt-injection-patterns]])
- **Controlled RAG over raw RAG** — use system-side checkpoints, metadata filtering, and human-in-the-loop to refine retrieved context before it hits the agent ([[concepts/controlled-rag]])
- **Specialized sub-agents save context** — each sub-agent has its own smaller static prompt and focus area, saving tokens vs. a monolithic agent ([[concepts/sub-agents]])
- **Tools need stop conditions** — without input constraints and sequential ordering, tools loop indefinitely searching for "enough" ([[concepts/sequential-tool-calling]], [[concepts/tool-use]])
- **Working vs. persistent memory** — dividing memory into what's needed now vs. what's learned over time; an ADHD analogy: low working memory makes agents focus on solving rather than finding solutions ([[concepts/agent-memory]])
- **Vanity metrics deceive** — metrics must tie to user satisfaction and business outcomes, not just accuracy/precision; multi-layered evaluation with an evaluator agent ([[concepts/agent-evals]])
