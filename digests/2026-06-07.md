# Digest 2026-06-07

The inaugural digest of the Raw Knowledge ontology. One source ingested so far — an Anthropic
applied-AI workshop on building [[concepts/long-running-agents]] — which seeded the entire
first concept cluster (16 concepts, 5 entities).

## Sources Covered

- [[sources/2026-06-07-anthropic-workshop-build-agents-that-run-for-hours|Anthropic Workshop: Build Agents That Run for Hours]]
  — Ash Prabaker & Andrew Wilson (AI Engineer conf). A history tour of the past year of
  Claude/Claude Code releases, followed by the current state-of-the-art harness pattern.
  Core thesis: long-horizon autonomous agents are made possible not by the model alone but by
  an evolving [[concepts/agent-harness]] — and today's best pattern is an adversarial
  [[concepts/generator-evaluator-pattern]] with separate context windows and file-based state.

## Emerging Themes

### 1. The harness is the product, and it is disposable
The strongest through-line is that the [[concepts/agent-harness]] **co-evolves** with the
model: every component exists to patch a current model weakness, and is meant to be deleted
once that weakness is trained into the weights. Concrete evidence — the team dropped fresh
context windows and sprint decomposition (critical for Opus 4.5) once Opus 4.6 held long
coherent sessions via [[concepts/compaction]]. The lesson framed sharply: *"our harness
wasn't wrong — it was right for the previous model; the frontier moved."*

### 2. Adversarial separation beats self-reflection
A cluster of concepts all point the same direction: don't ask one agent to do everything.
- [[concepts/generator-evaluator-pattern]] — a harsh critic with its own context window
- [[concepts/planner-generator-evaluator-architecture]] — adds a high-level planner (PM/IC/QA)
- [[concepts/sub-agents]] / [[concepts/agent-teams]] — the primitives that make it economical
The key insight is asymmetric: tuning a standalone critic to be harsh is *tractable*; tuning a
builder to be self-critical is *not*. Self-evaluation is explicitly called "a trap."

### 3. Context is the master constraint
[[concepts/context-window-management]] is the umbrella over a surprising number of the
year's innovations: [[concepts/compaction]], [[concepts/skills-progressive-disclosure]],
[[concepts/programmatic-tool-calling]], and using the [[concepts/file-system-state]] as
external memory. The failure mode it fights — [[concepts/context-rot]] / context anxiety — is
framed as *temporary*, a quirk of today's models.

### 4. Verification means *using* the thing
[[concepts/verification-loop]] is what separates "looks done" from "is done." The evaluator
drives a live app with [[entities/playwright]], catching bugs (route ordering, key handlers)
that unit tests/CI miss. The same philosophy extends to subjective quality via the
[[concepts/design-taste-rubric]] — "you can grade taste if you write down a strong opinion."

### 5. The human skill is reading traces
Repeatedly, the speakers reduce harness-building to [[concepts/reading-traces]] by hand and
*empathizing* with the model (the "browse with eyes closed" exercise). Observability for
ultra-long runs is named as unsolved greenfield.

## Contradictions & Tensions

> [!contradiction] Compaction vs. coherence (internal to the source)
> The talk embraces single-session + [[concepts/compaction]] (replacing fresh context
> windows) yet warns "compaction ≠ coherence — lossy summaries drift." Resolution offered:
> it's model- and use-case-dependent; Opus 4.6 holds coherence well enough, but harder work
> still benefits from structured hand-offs. **Watch for future sources that take a firm side.**

> [!contradiction] Ralph Loop — dead or still useful?
> [[concepts/ralph-loop]] is positioned as superseded (no contract negotiation, patches
> instead of pivots), but an audience "smart-zone/dumb-zone" argument defends task-slicing to
> keep the model sharp. The speakers concede it "still has its place depending on use case."

2. **"Why critics are cheaper than self-critics"** — Write up the asymmetry at the heart of

   tractable while a self-critical builder is not? Connect to the GAN analogy and the
